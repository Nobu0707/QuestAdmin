# CLAUDE.md

## 1. Claudeの役割

このプロジェクトでは、Claudeは実装担当です。

Claudeの担当範囲:

- Javaコード実装
- Forge 1.20.1向け実装
- ビルド実行
- エラー修正
- リファクタリング
- READMEや設定ファイルの実装反映

Claudeが勝手に行ってはいけないこと:

- 仕様を勝手に拡張する
- 指定されていないPhaseの機能を先に実装する
- 大規模な設計変更を独断で行う
- 保存形式を無断で変更する
- mod idやpackage名を無断で変更する
- 経済MOD連携先を勝手に増やす
- GUIを指定前に作り始める
- 不要な演出機能を追加する
- Fabric向け実装を混ぜる

## 2. 最重要前提

このプロジェクトは **Forge 1.20.1用MOD** です。

以下は使用禁止です。

- Fabric API
- FabricLoader
- ModInitializer
- fabric.mod.json
- net.fabricmc 系 import
- FabricのCommand API
- FabricのNetworking API
- FabricのScreenHandler前提の設計

Forge 1.20.1の前提:

- Minecraft 1.20.1
- Forge 47.x 系
- Java 17
- `@Mod("questadmin")`
- `mods.toml`
- Forge EventBus
- `RegisterCommandsEvent`
- Forge Menu / Screen
- Forge SimpleChannel

## 3. 作業開始時のルール

作業を始める前に、必ず以下のファイルを確認してください。

1. `SPEC.md`
2. `ROADMAP.md`
3. `CLAUDE.md`

その後、ユーザーから指定されたPhaseだけを実装してください。

## 4. 開発の最優先事項

優先順位は以下とします。

1. `./gradlew build` が成功すること
2. Forge 1.20.1の構成として正しいこと
3. 既存機能を壊さないこと
4. 仕様範囲を守ること
5. データ保存で事故を起こさないこと
6. 報酬支払いで事故を起こさないこと
7. コードを読みやすく保つこと

## 5. 実装範囲の制限

各作業では、指定されたPhaseの内容だけを実装してください。

たとえば Phase 1.5 では、以下を実装・確認してよいです。

- Forge向けメインMODクラスの確認
- `mods.toml` の確認
- `build.gradle` の確認
- Fabric参照の除去
- configディレクトリ取得処理の調整
- `QuestStorage` のForge向け調整

Phase 1.5 で以下を実装してはいけません。

- GUI
- Menu
- Screen
- SimpleChannel
- コマンド
- 経済MOD連携
- アイテム消費処理
- 報酬支払い処理
- プレイヤー進行状況保存
- 村人/NPC連携
- 新しいクエスト種別

## 6. ビルド確認

実装後は必ず以下を実行してください。

```bash

```

修正後、再度以下を実行してください。


ビルドが成功するまで、指定範囲内で修正してください。

## 8. コード設計方針

### 8.1 データ構造

クエスト定義とプレイヤー進行状況は分離してください。

```text
QuestDefinition
- クエストそのものの設定

PlayerQuestState
- プレイヤーごとの進行・完了状態
```

この2つを混ぜないでください。

### 8.2 経済MOD連携

経済MOD連携は必ず `EconomyBridge` を経由してください。

直接、各所から経済MOD APIを呼ばないでください。

### 8.3 保存処理

保存処理は専用クラスに分離してください。

想定:

```text
QuestStorage
PlayerQuestStorage
```

コマンドクラスやGUIクラスに保存処理を直接詰め込まないでください。

### 8.4 GUI処理

GUIは後のPhaseで実装します。

Forge 1.20.1では、GUIはMenu / Screenを使って実装します。

GUI実装時も、画面表示とクエスト処理を分離してください。

```text
Menu / Screen
- 表示と入力

QuestService
- クエスト処理

QuestStorage
- 保存処理
```

### 8.5 ネットワーク通信

クライアントGUIからサーバー側処理へ送信する必要がある場合は、ForgeのSimpleChannelを使います。

ただし、SimpleChannelはGUI実装Phaseまで追加しないでください。

## 9. 安全性に関するルール

以下の事故を起こさないようにしてください。

- アイテムだけ消費されて報酬が支払われない
- 報酬が二重支払いされる
- 無効化されたクエストを完了できる
- 削除済みクエストを完了できる
- JSON読み込み失敗でサーバーがクラッシュする
- サーバー再起動でクエスト進行状況が消える

特に、報酬支払い処理は後のPhaseで慎重に実装します。

## 10. JSON方針

MVPでは以下の2ファイルを使用します。

```text
config/questadmin/quests.json
config/questadmin/player_quests.json
```

JSONの読み込みに失敗した場合は、サーバーをクラッシュさせず、ログにエラーを出してください。

ただし、不正なデータを無理に保存し直して上書きしないでください。

## 11. 命名方針

仮のmod id:

```text
questadmin
```

仮のpackage:

```text
net.nobu0707.questadmin
```

ただし、既存プロジェクトに異なるpackageがある場合は、既存の構成を優先してください。

特に、以下はユーザー確認なしに追加しないでください。

- Webサーバー関連
