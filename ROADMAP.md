# QuestAdmin ROADMAP

## 開発方針

このプロジェクトでは、仕様を小さなPhaseに分割して開発します。

各Phaseでは、指定された範囲だけを実装します。
勝手に先のPhaseの機能を実装しません。

本プロジェクトは **Forge 1.20.1** 向けです。

Fabric向けの実装は行いません。

各Phaseの完了条件は、原則として以下を満たすことです。

- `./gradlew build` が成功する
- Forge 1.20.1の構成として破綻していない
- 既存機能を壊さない
- 変更内容を説明できる
- 仕様外の拡張をしない

## Phase 1.5: Forge 1.20.1整合確認

### 目的

Phase 1で実装済みのデータ構造と保存処理が、Forge 1.20.1環境に正しく適合しているか確認し、必要最小限の修正を行います。

### 実装・確認するもの

- `build.gradle` がForge 1.20.1前提になっているか確認
- Java 17前提になっているか確認
- `mods.toml` が存在するか確認
- mod id が `questadmin` になっているか確認
- メインMODクラスが `@Mod("questadmin")` になっているか確認
- Fabric関連のimportや依存が残っていないか確認
- `QuestStorage` がForge環境で `config/questadmin/quests.json` を扱えるか確認
- `quests.json` が存在しない場合、サンプルクエストを生成できるか確認
- `./gradlew build` が成功するか確認

### 実装してよいもの

- Forge用メインMODクラスの軽微な修正
- configディレクトリ取得処理のForge向け修正
- 不要なFabric参照の削除
- `mods.toml` / `build.gradle` の軽微な修正
- Phase 1で作った `QuestStorage` のForge向け調整

### 実装しないもの

- GUI
- Menu
- Screen
- ネットワーク通信
- SimpleChannel
- コマンド
- 経済MOD連携
- アイテム消費処理
- 報酬支払い処理
- プレイヤー進行状況保存
- 村人/NPC連携
- 新しいクエスト種別

### 完了条件

- `./gradlew build` が成功する
- Fabric関連のimportや依存が残っていない
- Forge 1.20.1用MODとして起動できる構成になっている
- `config/questadmin/quests.json` の保存方針がForge向けになっている
- Phase 1の範囲を超えた機能を追加していない
